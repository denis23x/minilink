# Internationalization (i18n)

## Current Status

This project has **no i18n setup**. The app is English-only.

- No `next-intl`, `i18next`, or any translation library is installed
- The root `<html>` element has `lang="en"` hardcoded in `src/app/layout.tsx`
- There is no `[locale]` route segment — the only route is `/`
- All user-facing strings are hardcoded in components

## If i18n Is Added in the Future

Recommended stack:

| Tool        | Purpose                          |
| ----------- | -------------------------------- |
| `next-intl` | Routing + message loading        |
| JSON files  | `messages/{locale}.json`         |

Steps to introduce:
1. Install `next-intl`
2. Add `[locale]` route segment wrapping `src/app/`
3. Configure `src/i18n/routing.ts` and `src/i18n/request.ts`
4. Add locale constants and message files
5. Replace all hardcoded strings with `t()` calls
6. Update `src/middleware.ts` to handle locale routing before `linkMiddleware`
